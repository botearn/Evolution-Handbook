# Knowledge Review

**状态：Draft**

Handbook 需要定期删、降级和复审。否则 Standard 会过时，Draft 会越堆越多。

## 复审对象

优先复审：

- `Standard` 和 `Adopted`
- 安全、权限、结算、生产操作相关文档
- 跨仓 Contract
- 绑定外部工具或 API 的指南
- 长期无人引用的 Draft

## 触发条件

出现以下情况应复审：

- 代码或生产行为和文档冲突
- Owner、项目边界或仓库归属变化
- 外部工具、API、模型能力变化
- 同一规则出现两个真源
- 规则连续两次没有产生预期效果

## 周期

建议节奏：

| 内容 | 复审频率 |
|---|---|
| Standard / Adopted | 每季度 |
| 高风险 Contract | 每次相关变更后 |
| 工具指南 | 产品能力变化后 |
| Draft | 半年无人使用则删除、合并或降级 |

## 处理方式

复审结论只选一种：

- 保留：仍然有效。
- 更新：事实变化，但方向仍对。
- 降级：从 Standard / Adopted 回到 Draft。
- 废弃：标记 Deprecated，并说明替代方案。
- 删除：无引用、无证据、无长期价值。

## 完成标准

- [ ] 状态已更新
- [ ] Owner 或待指定已写清
- [ ] 证据或最后核对日期已补充
- [ ] 索引和 README 已同步
- [ ] 被废弃内容有替代路径或删除理由
