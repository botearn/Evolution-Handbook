# Documentation Lifecycle

**状态：Draft**

## 目标

让文档随系统演进，而不是在创建后逐渐失真。

## 生命周期

```text
Candidate -> Draft -> Adopted -> Standard
                    \-> Deprecated -> Superseded
```

- `Candidate`：有价值的观察，尚未形成完整做法。
- `Draft`：结构完整，正在项目中试用。
- `Adopted`：至少一个项目使用并有验证证据。
- `Standard`：相关项目默认采用，例外需要说明。
- `Deprecated`：不再推荐，保留迁移背景。
- `Superseded`：由另一文档明确替代。

## 维护要求

每次修改系统边界、共享契约或操作流程时，检查：

- 代码和可执行资产是否仍是真实行为的真源
- README 和目录索引是否需要更新
- 引用路径、命令和外部链接是否仍有效
- 状态是否反映真实采用情况
- 临时兼容路径是否已达到退出条件

## 复审优先级

优先复审：

1. 涉及安全、权限、结算和生产操作的文档。
2. 两个以上仓库共同依赖的契约。
3. 绑定外部产品版本的工具指南。
4. 长期无人引用或没有 Owner 的文档。

## 完成条件

一次文档变更完成时：

- 所在目录索引已同步
- 本地链接和代码块检查通过
- 事实、建议与假设已区分
- 没有引入新的重复真源
- 相关项目文档已同步
