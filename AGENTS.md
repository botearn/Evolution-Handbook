# Evolution Handbook Agent Guide

本仓库是同事和 Agent 共同使用的组织级协作真源。

它的目标不是保存所有信息，而是让人和 Agent 能快速找到：当前任务该看哪里、谁是真源、风险在哪里、完成后如何验证和沉淀。

## Agent 入口

### 我能自主执行什么

- 阅读和检索本仓库公开文档，整理摘要、差异和候选归属。
- 新增或修改 Draft、README、索引、模板和低风险文档内容，并同步相关链接。
- 在本地分支提交文档变更，前提是工作区状态清楚且不覆盖他人未说明的修改。
- 运行本地只读检查、链接检查、格式检查和测试命令，并如实报告结果。

### 我不能做什么

- 写入密钥、账号、客户数据、内部访问地址、本机路径或生产凭据。
- 未经明确授权执行发布、权限、结算、生产数据、外部正式消息等高影响操作。
- 绕过审计、隐藏变更来源、伪造验证结果或把未经验证的内容包装成 Standard。
- 把其他仓库整篇文档复制进 Handbook；应优先链接真源并沉淀跨项目结论。

### 必须 dry-run + 人审的操作

- 发布、部署、权限变更、结算配置、生产数据修改。
- 对外部正式渠道发送消息、公告或代表团队作出承诺。
- 破坏性跨项目契约变更、目录结构迁移、批量重命名或删除长期文档。
- 任何影响范围、可逆性或授权边界不清楚的操作。

### 项目契约在哪里

- 内容归属：`WORKSPACE.md`
- 真源和 Owner：`SOURCE_OF_TRUTH.md`
- 风险分级：`ai-native/governance/risk-and-automation-levels.md`
- Agent 系统结构：`ai-native/agent-system/README.md`
- 项目契约：`projects/<project>/contracts/`
- 仓库本地实现约束：对应代码仓库的 `AGENTS.md`、`docs/`、Schema 和可执行资产

### 不确定时默认行为

- 先读路由和真源，不凭记忆判断归属。
- 选择更低风险、更可逆、更容易审查的路径。
- 先形成 Draft 或候选内容，不直接写成组织默认规则。
- 先展示 Diff、Dry Run 或目标资源，再请求人类确认。
- 事实与文档冲突时，以真实系统为准，并记录需要修正的文档。

## Context 加载顺序

1. 读当前任务的人类指令，确认是否有明确禁止、授权或交付格式。
2. 读 `WORKSPACE.md`，判断内容应属于组织、项目还是仓库层级。
3. 读目标目录的 `README.md`，确认本目录收什么、不收什么。
4. 读 `SOURCE_OF_TRUTH.md`，确认 Owner、真源和是否应回到代码仓库。
5. 若任务涉及项目边界或跨仓不变量，读 `projects/<project>/contracts/` 和相关 decisions。
6. 若任务涉及 Agent 自主执行、权限、发布、外部消息或生产影响，读 `ai-native/governance/risk-and-automation-levels.md`。
7. 若任务涉及 Agent 指令、Skill、工具或上下文机制，读 `ai-native/agent-system/` 和 `ai-native/context-system/`。
8. 最后再全仓搜索，避免用关键词搜索替代路由判断。

## 工作原则

- 默认使用中文编写正文；代码、协议名和业内固定术语可保留英文。
- 查找或新增内容前先读 `WORKSPACE.md`，按路由确定层级。
- 先判断内容属于组织、项目还是仓库层级，再决定写入位置。
- 不复制其他仓库中的整篇文档；优先链接，并在这里记录跨项目结论。
- 不在公共 Handbook 中写入密钥、账号、客户数据、内部访问地址或生产凭据。
- 原则写“为什么”，模式写“何时采用”，Playbook 写“如何执行”，案例写“发生了什么”。
- 未经验证的想法应明确标记为 Draft，不得包装成 Best Practice。
- 修改跨项目契约时，列出受影响的项目、仓库、负责人和验证方式。
- 帮助团队成员降低贡献门槛；不确定归属时先形成 Draft，不要求贡献者预先理解全部信息架构。
- 修改入口、协作流程或目录结构时同步检查根 `README.md` 和 `CONTRIBUTING.md`。

## 文档层级

冲突时按以下顺序处理：

1. 安全、合规和明确的人类指令
2. 仓库本地可执行规则
3. 项目级契约与决策
4. Handbook 组织级通用实践

更具体的规则可以覆盖通用规则，但必须记录覆盖原因。

## 完成标准

- 文档放在正确层级
- 标题和链接可读
- 事实、建议和待验证假设被明确区分
- 没有复制出第二份难以同步的真源
- 新增项目或仓库时同步更新相关项目文档
- 新增文档时更新所在目录的 `README.md`
