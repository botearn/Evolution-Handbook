# Evolution Handbook Agent Guide

本仓库是同事和 Agent 共同使用的组织级协作真源。

它的目标不是保存所有信息，而是让人和 Agent 能快速找到：当前任务该看哪里、谁是真源、风险在哪里、完成后如何验证和沉淀。

## 工作原则

- 默认使用中文编写正文；代码、协议名和业内固定术语可保留英文。
- 查找或新增内容前先读 `WORKSPACE.md`，按路由确定层级。
- 先判断内容属于组织、项目还是仓库层级，再决定写入位置。
- 不复制其他仓库中的整篇文档；优先链接，并在这里记录跨项目结论。
- 不在公共 Handbook 中写入密钥、账号、客户数据、内部访问地址或生产凭据。
- 原则写“为什么”，模式写“何时采用”，Playbook 写“如何执行”，案例写“发生了什么”。
- 未经验证的想法应明确标记为 Draft，不得包装成 Best Practice。
- 修改跨项目契约时，列出受影响的项目、仓库、负责人和验证方式。
- 帮助团队成员降低贡献门槛；不确定归属时先形成 Draft，不要求贡献者预先理解全部信息架构。
- 修改入口、协作流程或目录结构时同步检查根 `README.md` 和 `CONTRIBUTING.md`。

## 文档层级

冲突时按以下顺序处理：

1. 安全、合规和明确的人类指令
2. 仓库本地可执行规则
3. 项目级契约与决策
4. Handbook 组织级通用实践

更具体的规则可以覆盖通用规则，但必须记录覆盖原因。

## 完成标准

- 文档放在正确层级
- 标题和链接可读
- 事实、建议和待验证假设被明确区分
- 没有复制出第二份难以同步的真源
- 新增项目或仓库时同步更新 `registry/`
- 新增文档时更新所在目录的 `README.md`
