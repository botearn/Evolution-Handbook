# Onboarding Quickstart

新成员不需要第一天读完整个 Handbook。先读能帮助你开始协作的几页。

## 第一天：知道从哪里开始

按顺序读：

1. [README](README.md)：了解 Handbook 是什么。
2. [Workspace Routing](WORKSPACE.md)：知道内容放哪里。
3. [Source of Truth](SOURCE_OF_TRUTH.md)：知道冲突时信谁。
4. [参与贡献](CONTRIBUTING.md)：知道怎么提小改动。

完成后，你应该能回答：

- 我当前要找的内容属于组织、项目还是仓库？
- 文档和代码冲突时先看哪里？
- 我发现错别字、断链或过期描述时怎么改？

## 第一个贡献

建议从低风险改动开始：

- 修 typo
- 修断链
- 补 README 导航
- 给 Draft 增加限制或反例
- 补一个真实小案例

小改动不需要先理解全部架构。能说明“改了什么、为什么、如何检查”就够。
