Onboarding Quickstart ​

新成员不需要第一天读完整个 Handbook。先读能帮助你开始协作的几页。

第一天：知道从哪里开始 ​

按顺序读：

  1. README：了解 Handbook 是什么。
  2. Workspace Routing：知道内容放哪里。
  3. Source of Truth：知道冲突时信谁。
  4. 参与贡献：知道怎么提小改动。

完成后，你应该能回答：

  • 我当前要找的内容属于组织、项目还是仓库？
  • 文档和代码冲突时先看哪里？
  • 我发现错别字、断链或过期描述时怎么改？

第一周：按角色补读 ​

做工程 ​

  • ARTI 项目入口
  • ARTI 架构
  • 跨仓库契约
  • 跨仓库变更

做产品 / 项目协作 ​

  • 项目目录
  • ARTI 决策索引
  • 文档生命周期

使用 Agent ​

  • AI Native
  • 验证优先
  • AI 工具实践
  • 可回看的 Agent 工作台

第一个贡献 ​

建议从低风险改动开始：

  • 修 typo
  • 修断链
  • 补 README 导航
  • 给 Draft 增加限制或反例
  • 补一个真实小案例

小改动不需要先理解全部架构。能说明“改了什么、为什么、如何检查”就够。
