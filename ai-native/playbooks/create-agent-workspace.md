# Create an Agent Workspace

**状态：Draft**

## 目标

让一个项目具备清晰的 Agent 规则、工作流真源和跨宿主兼容方式。

## 步骤

1. 审计项目中的 `AGENTS.md`、宿主文件、Skills 和散落 Prompt。
2. 将平台无关规则整理到项目或仓库的 `AGENTS.md`。
3. 识别可复用工作流；只有稳定工作流才进入 `skills/`。
4. 统一 Skill 的名称、描述和触发方式。
5. 为需要支持的宿主生成薄 Bridge。
6. 检查 Bridge 能否指回真源，引用路径是否存在。
7. 用一个真实任务完成端到端验证。
8. 在项目文档中记录维护方式和负责人。

## 验收清单

- [ ] `AGENTS.md` 可以独立指导仓库工作
- [ ] 长期逻辑只有一个真源
- [ ] 宿主 Bridge 名称一致
- [ ] 没有悬空路径
- [ ] 权限与高风险动作有明确边界
- [ ] 至少完成一次真实任务验证
