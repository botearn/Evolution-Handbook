# Human-Agent Collaboration

**状态：Draft**

## 默认协作循环

```text
人类定义目标与边界
  → Agent 收集上下文并提出计划
  → Agent 执行可逆工作
  → 工具和测试验证结果
  → 人类处理高风险判断
  → 结果沉淀为规则、模板或案例
```

## 责任分配

人类更适合：

- 定义价值、优先级和不可接受后果
- 处理新颖、模糊和高影响判断
- 批准不可逆操作
- 对最终结果承担责任

Agent 更适合：

- 搜集和连接分散上下文
- 执行重复且可验证的步骤
- 生成候选方案并暴露权衡
- 维护索引、检查漂移和运行验证

确定性工具更适合：

- 路由、格式转换和数据校验
- 测试、Lint、类型检查和 Schema 验证
- 重试、调度和权限执行

## 每次工作应留下什么

不是每次任务都需要新增文档。只有出现可复用的新信息时，才沉淀为：

- 原则：长期稳定的判断
- 模式：重复问题的解决结构
- Playbook：可执行步骤
- 工具：可自动化的确定性操作
- 案例：包含结果和教训的真实证据
