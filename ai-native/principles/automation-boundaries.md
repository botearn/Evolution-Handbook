# Automation Boundaries

**状态：Draft**

自动化边界应由风险、可逆性和可验证性决定，而不是由技术上能否自动完成决定。

## 四级模型

| 级别 | Agent 权限 | 适用场景 |
|---|---|---|
| Assist | 提供分析和草稿 | 新领域、高不确定性 |
| Recommend | 给出建议，人类决定 | 有规则但影响较高 |
| Execute with approval | 准备动作，批准后执行 | 可验证但有外部影响 |
| Autonomous | 自动执行并记录 | 低风险、可逆、强验证 |

## 升级自动化的条件

- 输入边界稳定
- 失败模式已知
- 有自动验证
- 有审计记录
- 有停止和回滚机制
- 已通过真实案例验证

任一条件失效时，应降低自动化级别。
