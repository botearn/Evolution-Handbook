# Verification First

**状态：Draft**

Agent 产出的速度不能替代结果验证。工作流应在执行前定义完成标准，在执行后提供可检查证据。

## 原则

- 开始工作前定义成功条件。
- 确定性问题优先使用代码、测试和结构化工具验证。
- “没有报错”不等于“行为正确”。
- 测试应验证业务意图，而不只是覆盖代码路径。
- 未执行、跳过或无法验证的部分必须显式说明。

## 风险匹配

| 风险 | 最低验证要求 |
|---|---|
| 低 | 静态检查或人工快速复核 |
| 中 | 自动测试加关键路径复核 |
| 高 | 独立复核、审计记录和回滚方案 |
| 不可逆 | 明确的人类批准后执行 |

## 完成声明

Agent 声明“完成”时，应能回答：

- 改了什么
- 如何验证
- 哪些没有验证
- 剩余风险是什么
