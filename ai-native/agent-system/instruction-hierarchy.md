# Agent Instruction Hierarchy

**状态：Draft**

Agent 指令应按所有权和作用域分层，避免多个宿主各自维护一套长期逻辑。

## 推荐层级

```text
组织级原则
  ↓
项目级契约与例外
  ↓
仓库级 AGENTS.md
  ↓
目录级局部规则
  ↓
当前任务的人类指令
```

安全、合规和权限边界始终优先。其他冲突通常由更具体、离实现更近的规则覆盖。

## 文件职责

- `AGENTS.md`：平台无关、可执行的仓库规则
- `CLAUDE.md` 等宿主文件：仅保留宿主差异或指向公共真源
- `skills/`：可复用工作流的项目真源
- 用户目录下的 Skill：薄 Bridge，不维护长期正文

## 要求

- 指令应可操作，避免只写价值口号。
- 仓库规则必须包含真实命令和关键约束。
- 已过期规则应删除或标记，不依赖 Agent 猜测。
- 指令中引用的文件和命令应定期检查是否仍存在。
