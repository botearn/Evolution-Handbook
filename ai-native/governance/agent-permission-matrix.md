# Agent Permission Matrix

**状态：Draft**

这张表回答：Agent 哪些事可以自己做，哪些必须先给人看，哪些不要做。

## 权限矩阵

| 类型 | Agent 可做什么 | 要求 |
|---|---|---|
| 读文档、读代码、搜索 | 自主执行 | 不读取密钥、私密数据或无关本机文件 |
| 改 README、typo、链接、索引 | 自主修改，提交前验证 | 跑相关检查，说明改了什么 |
| 改业务代码或契约 | 先计划，再修改 | 说明影响范围和验证方式 |
| 写外部系统、发消息、改权限 | dry-run + 人批准 | 先展示将要执行的内容 |
| 发布、结算、生产数据修复 | 人批准 + 可回滚方案 | 留记录，执行后回读 |
| 删除数据、绕过审计、隐藏行为 | 禁止 | 不接受“为了省事”的例外 |

## 默认规则

- 读可以宽，写要窄。
- 可逆动作比不可逆动作低风险。
- 本地草稿低风险，外部系统高风险。
- 影响钱、权限、生产数据、正式对外消息时，默认需要人批准。
- Agent 不能把“没报错”当成完成。

## Dry-run 应包含

- 要改什么
- 会影响谁
- 实际命令或请求内容
- 回滚方式
- 验证方式

## 完成声明

Agent 结束时应说明：

- 做了什么
- 跑了什么检查
- 哪些没验证
- 还剩什么风险
