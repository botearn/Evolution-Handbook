# AI Risk and Automation Levels

**状态：Draft**

## 目标

让 Agent 的自主程度与错误影响匹配，而不是按工具或个人偏好统一放开。

## 风险等级

| 等级 | 示例 | 默认自动化 |
|---|---|---|
| R0 只读低敏 | 公共文档检索、代码解释 | 可自动 |
| R1 可逆局部写入 | 草稿、测试、格式化、局部分支修改 | 自动执行并验证 |
| R2 共享但可恢复 | PR、共享文档、测试环境配置 | 执行前确认范围 |
| R3 高影响写入 | 发布、权限、结算、生产数据、外部消息 | 人工明确批准 |
| R4 不可接受 | 泄露凭证、绕过审计、未授权数据访问 | 禁止 |

## 判断维度

- 数据敏感度
- 影响人数和系统范围
- 是否可逆
- 是否有独立验证
- Agent 当前身份和权限
- 外部输入是否可能包含 Prompt Injection

任一维度进入更高等级时，采用更严格等级。

## 权限矩阵

| 操作类型 | 默认等级 | 自主 | Dry-run + 人审 | 禁止 | 说明 |
|---|---:|---:|---:|---:|---|
| 只读检索、摘要、目录导航 | R0 | ✅ |  |  | 保留来源和引用路径。 |
| Draft 文档、README、索引和模板修改 | R1 | ✅ |  |  | 修改后展示 Diff，并运行可用检查。 |
| 本地测试、格式化、链接检查 | R1 | ✅ |  |  | 不应修改外部资源。 |
| 代码修改或 PR 草稿 | R2 |  | ✅ |  | 需要说明范围、Diff 和验证结果。 |
| 共享文档发布、目录迁移、批量删除 | R2/R3 |  | ✅ |  | 影响范围越大，按 R3 处理。 |
| 生产发布、部署、回滚 | R3 |  | ✅ |  | 需要明确授权、审计记录和写后回读。 |
| 权限、结算、生产数据变更 | R3 |  | ✅ |  | 默认最小权限和双重确认。 |
| 发送外部正式消息或组织承诺 | R3 |  | ✅ |  | 先展示完整发送内容和目标渠道。 |
| 泄露凭证、绕过审计、未授权访问 | R4 |  |  | ❌ | 直接拒绝或阻止。 |

## 控制方式

- R0：限制读取范围，保留来源。
- R1：在 Sandbox 中执行，运行机器可读验证。
- R2：展示 Diff、Dry Run 或目标资源，确认后写入。
- R3：最小权限、双重确认、审计记录和写后回读。
- R4：由 Policy、Hook、Rules 或权限系统直接阻止。

## 例外

例外必须记录：

- 业务理由
- 时间范围
- 授权人
- 补偿控制
- 自动失效或复审日期

“为了减少确认次数”不能单独构成放宽权限的理由。
