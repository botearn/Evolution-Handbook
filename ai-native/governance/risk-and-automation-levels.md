# AI Risk and Automation Levels

**状态：Draft**

让 Agent 的自主程度与错误影响匹配。任一维度进入更高风险时，采用更严格等级。

## 风险等级

| 等级 | 示例 | 默认自动化 |
|---|---|---|
| R0 只读低敏 | 公共文档检索、代码解释 | 可自动 |
| R1 可逆局部写入 | 草稿、测试、格式化、局部分支修改 | 自动执行并验证 |
| R2 共享但可恢复 | PR、共享文档、测试环境配置 | 执行前确认范围 |
| R3 高影响写入 | 发布、权限、结算、生产数据、外部消息 | 人工明确批准 |
| R4 不可接受 | 泄露凭证、绕过审计、未授权访问 | 禁止 |

## 权限矩阵

| 操作 | 自主 | Dry-run + 人审 | 禁止 |
|---|---:|---:|---:|
| 只读检索、摘要、目录导航 | ✅ |  |  |

| 本地测试、格式化、链接检查 | ✅ |  |  |

| 共享文档发布、目录迁移、批量删除 |  | ✅ |  |
| 生产发布、部署、回滚 |  | ✅ |  |
| 权限、结算、生产数据变更 |  | ✅ |  |
| 外部正式消息或组织承诺 |  | ✅ |  |
| 泄露凭证、绕过审计、未授权访问 |  |  | ❌ |

## 判断维度

- 数据敏感度
- 影响范围
- 是否可逆
- 是否有独立验证
- Agent 当前身份和权限
- 外部输入是否可能包含 Prompt Injection

## 例外

例外必须记录业务理由、时间范围、授权人、补偿控制和复审日期。减少确认次数不能单独构成放宽理由。
