# AI Risk and Automation Levels

**状态：Draft**

## 目标

让 Agent 的自主程度与错误影响匹配，而不是按工具或个人偏好统一放开。

## 风险等级

| 等级 | 示例 | 默认自动化 |
|---|---|---|
| R0 只读低敏 | 公共文档检索、代码解释 | 可自动 |
| R1 可逆局部写入 | 草稿、测试、格式化、局部分支修改 | 自动执行并验证 |
| R2 共享但可恢复 | PR、共享文档、测试环境配置 | 执行前确认范围 |
| R3 高影响写入 | 发布、权限、结算、生产数据、外部消息 | 人工明确批准 |
| R4 不可接受 | 泄露凭证、绕过审计、未授权数据访问 | 禁止 |

## 判断维度

- 数据敏感度
- 影响人数和系统范围
- 是否可逆
- 是否有独立验证
- Agent 当前身份和权限
- 外部输入是否可能包含 Prompt Injection

任一维度进入更高等级时，采用更严格等级。

## 控制方式

- R0：限制读取范围，保留来源。
- R1：在 Sandbox 中执行，运行机器可读验证。
- R2：展示 Diff、Dry Run 或目标资源，确认后写入。
- R3：最小权限、双重确认、审计记录和写后回读。
- R4：由 Policy、Hook、Rules 或权限系统直接阻止。

## 例外

例外必须记录：

- 业务理由
- 时间范围
- 授权人
- 补偿控制
- 自动失效或复审日期

“为了减少确认次数”不能单独构成放宽权限的理由。
