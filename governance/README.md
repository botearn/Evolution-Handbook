# Governance

这里记录组织如何做出长期决策、分配责任并控制风险。

## 当前内容

- [Handbook 内容治理](handbook-content-policy.md)
- [仓库治理规则](repository-policy.md)

## 适合放在这里

- 决策权限与责任边界
- 仓库和项目的建立、归档与所有权规则
- 安全、隐私、合规和数据边界
- 组织级技术与 AI 风险分级
- 例外申请与复审机制

具体项目的产品决策和架构决策应放在 `projects/<project>/decisions/`。

## 当前缺口

- 组织级安全与数据分类
- 例外申请和定期复审机制
- 项目创建、暂停与归档流程
- 跨项目技术决策的批准边界

缺口只有在责任人和真实流程明确后才写成 Standard。
