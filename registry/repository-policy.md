# Repository Policy

**状态：Draft**

## 基本原则

- 一个仓库应有明确的项目归属、负责人和生命周期状态。
- 仓库中的可执行资产是实现事实的真源。
- 每个活跃代码仓库应有 `README.md` 和平台无关的 `AGENTS.md`。
- 宿主专属文件应保持轻量，不重复维护长期规则。
- 生产凭据不得进入仓库，包括公共与私有仓库。

## 最小登记信息

每个仓库应在 `registry/repositories.yaml` 中登记：

- 仓库标识和远端地址
- 所属项目
- 主要职责
- 生命周期状态
- 默认分支
- 文档与契约入口

## 归档

归档仓库前需要：

1. 标记替代方案或终止原因。
2. 迁移仍被依赖的契约与文档。
3. 更新项目仓库地图和 Registry。
4. 确认没有自动化仍向该仓库发布。
